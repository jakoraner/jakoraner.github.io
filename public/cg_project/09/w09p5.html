Comparison of Projection Shadows and Shadow Mapping

Introduction

In computer graphics, rendering realistic shadows enhances the depth and realism of a scene. Two common techniques for generating shadows are projection shadows and shadow mapping. In this document, I will compare these two methods by discussing their advantages and disadvantages, and provide code examples to illustrate how they are implemented.

Projection Shadows

Projection shadows involve projecting the silhouette of an object onto a surface from the light's perspective. This technique is relatively straightforward and involves transforming the object's geometry to create a shadow on a receiver surface, such as the ground.

Advantages of Projection Shadows

  • Simple Implementation: The mathematics behind projection shadows are straightforward, involving basic matrix transformations.
  • Performance Efficiency: Since it doesn't require additional rendering passes or complex calculations, it's computationally less intensive.

Disadvantages of Projection Shadows

  • Limited to Planar Surfaces: Projection shadows work best on flat surfaces. Projecting onto arbitrary or curved surfaces is more complex.
  • No Self-Shadowing: Objects cannot cast shadows onto themselves or onto other objects, reducing realism.
  • Single Light Source: Extending this technique to multiple light sources can be cumbersome.

Code Example

Below is a snippet from the projection shadow implementation. It shows how to compute the shadow matrix and apply it to the object's model matrix. 


// Compute the shadow projection matrix onto the ground plane
var groundY = -1.0; // y-coordinate of the ground plane
var epsilon = 0.001; // Small offset to prevent z-fighting
var yPlane = groundY + epsilon;

// Light position
var lightPosition = vec3(lightX, lightY, lightZ);

// Distance from light to the plane
var d = -(lightPosition[1] - yPlane);

// Translation matrices
var T_light = translate(lightPosition[0], lightPosition[1], lightPosition[2]);
var T_neg_light = translate(-lightPosition[0], -lightPosition[1], -lightPosition[2]);

// Projection matrix onto y = yPlane
var P_proj = mat4();
P_proj[3][1] = 1 / d;
P_proj[3][3] = 0;

// Shadow matrix
var shadowMat = mult(T_light, mult(P_proj, T_neg_light));

// Apply the shadow matrix to the object's model matrix
var modelViewMatrixShadow = mult(viewMatrix, mult(shadowMat, modelMatrixTeapot));

Shadow Mapping

Shadow mapping is a more advanced technique that uses depth textures to determine whether a fragment is in shadow. It involves rendering the scene from the light's point of view to create a shadow map (depth texture) and then using this map during the final rendering pass to apply shadows.

Advantages of Shadow Mapping

  • Supports Complex Geometries: Works well with arbitrary surfaces, including curved and uneven terrain.
  • Self-Shadowing: Objects can cast shadows onto themselves and onto other objects, increasing realism.
  • Multiple Light Sources: Can handle multiple lights by generating separate shadow maps for each.

Disadvantages of Shadow Mapping

  • Implementation Complexity: Requires multiple rendering passes and careful handling of coordinate transformations.
  • Artifacts Due to Precision: Limited depth buffer precision can cause issues like shadow acne or Peter Panning.
  • Performance Overhead: Generating and sampling shadow maps can be computationally intensive.

Code Example

Here's a fragment shader snippet that demonstrates how to use the shadow map to determine shadow visibility: 


precision mediump float;
varying vec4 v_PositionFromLight;
uniform sampler2D u_ShadowMap;

void main() {
    // Transform the fragment position into shadow map coordinates
    vec3 shadowCoord = v_PositionFromLight.xyz / v_PositionFromLight.w;
    shadowCoord = shadowCoord * 0.5 + 0.5;

    float visibility = 1.0;

    // Check if the fragment is within the shadow map bounds
    if (shadowCoord.x >= 0.0 && shadowCoord.x <= 1.0 &&
        shadowCoord.y >= 0.0 && shadowCoord.y <= 1.0) {

        // Get the depth from the shadow map
        float depthFromShadowMap = texture2D(u_ShadowMap, shadowCoord.xy).r;
        float depth = shadowCoord.z;

        // Apply a bias to prevent shadow acne
        if (depth > depthFromShadowMap + 0.005) {
            visibility = 0.5; // Fragment is in shadow
        }
    }

}

Conclusion

Projection shadows are a quick and efficient method for adding shadows to scenes with flat surfaces and a single light source. They are easy to implement but lack flexibility and realism in complex environments. On the other hand, shadow mapping offers greater realism by supporting self-shadowing and complex geometries. Although it is more complex and computationally intensive, shadow mapping is essential for modern graphics applications where realistic lighting is crucial. Understanding these techniques allows us to choose the appropriate method based on the needs of the application and the available resources.
